# Domain 2: Design Resilient Architectures

## Weight: 26% of Scored Content

This domain covers designing scalable, loosely coupled, highly available, and fault-tolerant architectures on AWS.

## Task Statements

| Task | Description |
|------|-------------|
| Task 2.1 | Design scalable and loosely coupled architectures |
| Task 2.2 | Design highly available and/or fault-tolerant architectures |

## Key AWS Services

- Amazon API Gateway, SQS, SNS, EventBridge
- Elastic Load Balancing (ALB, NLB)
- Amazon EC2 Auto Scaling
- AWS Lambda, Fargate, ECS, EKS
- Amazon Route 53
- Amazon RDS (Multi-AZ, Read Replicas), RDS Proxy
- Amazon S3, EBS, EFS
- AWS Step Functions
- AWS X-Ray

## Key Concepts

- Horizontal vs vertical scaling
- Loose coupling via queues and events
- Disaster recovery strategies (Backup/Restore, Pilot Light, Warm Standby, Active-Active)
- RPO and RTO
- Multi-AZ and multi-Region architectures
- Immutable infrastructure
